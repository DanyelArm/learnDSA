    **1. Create `HintGate.tsx` (per D-11, D-12, D-13, D-14):**

    Props:
    ```typescript
    interface HintGateProps {
      hints: string[]           // 3-tier hint array from exercise
      miniChallenges: MiniChallengeDTO[]  // mini-challenges for this exercise
      hintsUsed: number         // from UserProgress — how many tiers unlocked (0, 1, 2, or 3)
      topicId: number           // for the POST /api/topics/:id/hint call
      onHintUnlocked: (newHintsUsed: number) => void  // callback to update parent state
    }
    ```

    The component:
    - Renders a collapsible "Hints" section with 3 tiers
    - Tier labels: "Tier 1: Approach", "Tier 2: Pseudocode", "Tier 3: Key Snippet"
    - Tiers 1..hintsUsed are revealed (show the hint text from `hints[tierIndex]`)
    - The next locked tier shows a lock icon and "Prove Your Worth" button
    - Tiers beyond the next locked one show only lock icons (greyed out)
    - Clicking "Prove Your Worth" opens the MiniChallenge overlay with a randomly selected challenge from `miniChallenges` (per D-14: `miniChallenges[Math.floor(Math.random() * miniChallenges.length)]`)
    - On MiniChallenge success (`onSuccess`):
      1. Call `POST /api/topics/${topicId}/hint` with auth header (fetch from authStore token)
      2. On 200 response: call `onHintUnlocked(response.data.hintsUsed)` to update parent
      3. Process any `unlockedAchievements` from response (call processGamificationResponse if achievements returned)
    - On MiniChallenge cancel: close the challenge overlay, return to hint view
    - If `miniChallenges` array is empty or undefined: skip the gate, just reveal hints directly (graceful fallback for topics without mini-challenge content — topics 2-20 until Phase 5)
    - Medieval theme: hint tiers styled as scroll sections, lock icons as padlock emoji or SVG, gold "Prove Your Worth" button
